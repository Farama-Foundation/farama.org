--- title: Project Standard ---

List of Project Standards

Standardization

  • Support Gym and/or PettingZoo API at the newest version (alternative APIs in addition to these are fine)
  • Support all supported versions of Python (currently 3.7-3.10)

Good Science

  • Be fully reproducible given seeds and tested to confirm this
  • Deterministic builds for compiled libraries
  • Have hooks for hyperparameter tuning (learning libraries only)

Quality of Life

  • Good documentation website (if library is large enough)
  • Support for Linux and macOS (x86_64 and ARM)
  • Natively support PyPI

Testing

  • CI with API compliance tests
  • Pre-commit with black, flake8 and isort
  • Add type hinting with pyright (with testing at pre-commit)
  • Add docstrings for all reasonable functions (using pydocstyle and google style)

Long Term Support

  • Releases cut via GitHub action on GitHub release
  • Jordan is an owner on PyPI in case someone gets hit by a bus

Branding

  • Logo (in the same style as other projects)
  • Have documentation be at ___.farama.org (or have that address forward to documentation site), with Google analytics setup
  • Include readme.md on PyPI page

Licensing

  • MIT license of similar (Apache 2.0 is fine)

Future Standards

  • Expose vector API via envpool for C based environments
  • Poetry based requirements.txt autogeneration
  • Natively support Conda
